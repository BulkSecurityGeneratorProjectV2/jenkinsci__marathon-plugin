These credentials are used to generate a JSON Web Token (JWT) for authentication with DC/OS using Service Accounts.

The content of this credential is a JSON object with the following fields:

uid
Service Account ID.
Example: "jenkins-sa"
login_endpoint
The URL for the login end point of the target DC/OS cluster.
Example: "https://leader.mesos/acs/api/v1/auth/login"
scheme
The Algorithm to use with private_key.
Example: "RS256"
private_key
Private key to use for signing the JWT, which is verified by the corresponding key registered with DC/OS. Note the inclusion of "\n" to make the content JSON friendly.
Example: "-----BEGIN RSA PRIVATE KEY-----\nMIIEpAI[..snipped..]"

All other fields within the JSON object are ignored.
